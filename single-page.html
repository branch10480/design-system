Single Page Template

シンプルな 1 ページのパターン。

ヘッダーナビもサイドバーもない、コンテンツ + フローティングアンカーメニューだけの構成。 調査レポートや仕様書を 1 枚で公開するときのテンプレート。

Base Toshi Design System v0.3 構成 1 カラム · 760px Nav フローティングメニューのみ

概要

このページ自体がテンプレート。HTML をコピーして見出しと本文を差し替えれば、同じ見た目のレポートがすぐ作れる。

デザインルールは design-system.html 本体と完全に共通。 トークン（:root + dark ブロック）、JetBrains Mono、コードブロックのコピーボタンまで同じものが入っている。 違いはナビゲーションの構造だけ — このパターンには「ページ」がひとつしかないので、 ハッシュルーティングも左サイドバーも持たない。

ヒント

レポートの結論は最初のセクションに書く。読者の 8 割はスクロールしない前提で構成する。

フローティングメニュー

このページの唯一のナビゲーション。右端に固定表示され、スクロールに合わせて現在地がハイライトされる。

  1. 自動生成。各 .doc-section の id と data-title から JS がリンクを組み立てる。
  2. 素のアンカーリンク。ルーターを持たないので href="#section-id" がそのまま使える。URL 共有でセクションに直リンクできる。
  3. 狭い画面では右下のボタンに。1240px 未満ではフローティングボタン + ポップオーバーに切り替わる。

使える部品

このテンプレートに同梱しているのは、レポートでよく使う 4 種だけ：aside / コード / テーブル / ステップリスト。ほかが必要なら本体からコピーする。

Aside

注記

このテンプレートの aside は note / important / tip / warning の 4 種。

警告

機密情報（API キー・個人情報）を含めない。公開サーバーに置く前提で書く。

コード

// ホバーするとコピーボタンがフェード表示される
const sections = document.querySelectorAll('.doc-section[id]');
const links = [...sections].map(s =>
  `<a href="#${s.id}">${s.dataset.title}</a>`);

使い方

3 ステップで新しいレポートになる。

  1. このファイルをコピーする。single-page.html を新しい名前で保存。
  2. セクションを差し替える。id / data-title / 見出し / 本文を書き換える。セクションを増減すればメニューも追従する。
  3. ブラウザで確認して公開。ライト / ダーク両方を見ること。

重要

トークンの真実の源は本体の design-system.html。色や字形を変えたくなったら、まず本体を更新してからこのテンプレートへ反映する。
